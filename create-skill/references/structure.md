# Structure

Choose the body shape based on how much variation the task allows.

## Rigid Skill

Use a rigid structure when skipping a step could break the workflow.

- Use numbered steps.
- Use checklists.
- Use concrete commands.
- Keep the sequence explicit.

## Flexible Skill

Use a flexible structure when the agent needs judgment.

- Use priorities.
- Use decision criteria.
- Use examples.
- Keep the rules clear without forcing a single path.

## Progressive Disclosure

Use three levels:

1. Frontmatter for trigger metadata.
2. `SKILL.md` for the core workflow.
3. `references/` for details, examples, and edge cases.

## Good Body Practices

- Write in imperative voice.
- Keep the body short enough to load quickly.
- Put only core instructions in the main file.
- Move long examples and optional guidance into references.
